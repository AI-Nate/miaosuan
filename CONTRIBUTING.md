# Contributing

The most valuable contribution is a **new doctrine pack**. Spec: [`doctrines/README.md`](doctrines/README.md).

Wanted packs: `stoic/` · `munger/` · `boyd-ooda/` · `clausewitz/` · surprise us.

## Ground rules

1. **Lenses must have sharp triggers.** We review packs primarily on the quality of 适用场景 — patterns an agent can actually match against real events. "When facing difficulty" gets your PR closed with love.
2. **One worked example required.** Every new pack ships with one fictional worked review in `examples/` showing 2–3 of its lenses firing.
3. **Public-domain or properly licensed sources only.** Quote briefly; distill, don't reproduce.
4. **No real-company data in examples.** All examples are fictional. PRs containing real names, real metrics, or anything resembling credentials are rejected.
5. **Plain language.** If a lens needs a glossary, rewrite it.

## Non-doctrine contributions

Bug fixes to templates/SKILL.md, setup guides for other harnesses, and translations are all welcome. Keep the core loop unchanged (GATHER → INTERPRET → UPDATE → ESCALATE → REPORT) — that's the product.
